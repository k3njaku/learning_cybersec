First Blood: Fetching the Page

Set up the kitchen. Install the tools. Write 10 lines of Python that download a real website.
By the end of this chapter, your script will print the textilepages.com HTML to your screen. First dopamine hit. 🎯

"A scraper is just a very, very polite HTTP request."

Chapter 02.1 // Mental model

🧠 What a "fetch" actually is

Forget "scraping" for a second. You're just making the exact same request that your browser makes — except from a Python script instead of Chrome.

📝 Mailing a letter analogy

A browser is a mail carrier. You type a URL, it writes a letter to that server saying "please send me the /textile-companies page", mails it, and brings the response back.

Python's requests library is your personal mail carrier. Same letters, same delivery, same response. It just doesn't bother rendering the page for you — it hands you the raw envelope.

That "raw envelope" contents = the HTML source you saw in recon.

📤 What we send

GET /textile-companies HTTP/1.1
Host: textilepages.com
User-Agent: Mozilla/5.0 ...

A little text blob that says "hey, give me this page." That's a request.

📥 What we get back

HTTP/1.1 200 OK
Server: cloudflare
Content-Type: text/html

<html>...145 KB of HTML...</html>

A status code, some headers, and the page's HTML body. That's a response.

setup the kitchen
Chapter 02.2 // Setup

📦 Step 1 — Kitchen setup (venv + pip)

Before we cook, we set up a clean kitchen. In Python, that means a virtual environment.

📝 Why venv?

Every Python project is a kitchen. Without venv, you'd dump every project's knives into one drawer — chaos. Project A needs requests 2.25, Project B needs requests 2.31, now they break each other.

venv = a private kitchen per project. Pros NEVER skip this.

🐚 Run these in your terminal — one at a time

# Go to your working folder (the "forge") cd ~/Documents/DataScrapingLabs/textile-scraper # Activate the venv you created last time (the "kitchen") source venv/bin/activate # Your prompt should now show (venv) at the start — that's the kitchen lights turning on # Install the 3 tools we need from Python's app store (pip) pip install requests beautifulsoup4 lxml

🌐 requests

The HTTP library. Makes GET / POST requests. The mail carrier.

🥣 beautifulsoup4

HTML parser. We'll use this in Ch 03. Installing now so we're ready.

⚡ lxml

A fast C-based parsing engine BeautifulSoup uses under the hood. Optional but recommended.

🧠 Bash ninja trick: pip list after installing — shows every package in your current venv. which python tells you which Python you're using. If it says .../venv/bin/python, you're in the right kitchen. ✅
🧠 Brain Power
If you forget to run source venv/bin/activate and then run pip install requests — what goes wrong?

It installs requests to your global / system Python instead of the project's venv. On modern Ubuntu, this usually errors out with externally-managed-environment — which is actually helpful, it's Ubuntu saving you from yourself.

If you do succeed installing globally, weirder bugs appear later: your script might work on your machine but break on another, or two projects' dependencies might conflict.

👉 Rule: activate first, pip install second. Always.

write the code
Chapter 02.3 // Write the code

✍️ Step 2 — Your first scraper

Open ~/Documents/DataScrapingLabs/textile-scraper/scraper.py in VS Code. Type this — don't paste. Your fingers need to learn the shape.

# scraper.py — first fetch # Downloads textilepages.com and prints the first 500 characters of HTML. import requests # The target URL — this is where we did recon in Ch 01. URL = "https://textilepages.com/textile-companies" # A realistic User-Agent so Cloudflare treats us like a real browser, not a bot. HEADERS = { "User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 " "(KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36" } # Send the request and grab the response. response = requests.get(URL, headers=HEADERS) # Print basic info about what came back. print("Status:", response.status_code) print("Size :", len(response.text), "chars") print("--- first 500 chars of HTML ---") print(response.text[:500])

Then run it:

# Still inside your activated venv (you should see (venv) in prompt) python3 scraper.py

🎉 Expected output (roughly)

Status: 200 Size : 145703 chars --- first 500 chars of HTML --- <!DOCTYPE html> <html lang="en"> <head> <title> List of textile companies - Textile directory ... <meta name="description" content=" Textile companies directory ...
☕ You just scraped a website. Your Python script reached across the internet, asked Cloudflare for a page, and got 145,000 characters of HTML back. That's the entire mechanic of scraping in 9 lines of code. Everything from here is just refinement.

Chapter 02.4 // Decode it

🔬 Line by line — every piece explained

This is where you turn "it works" into "I understand it."

import requests

Pulls in the requests library you installed with pip. Without this line, Python doesn't know what requests means. Imports = "bring this tool into this file."

URL = "..."

A variable. A named box you can put stuff in and reuse. We put the target URL in a box so we can refer to it by name. ALL CAPS is a Python convention for "this is a constant — don't change it."

HEADERS = { ... }

A dictionary — a box that holds key-value pairs. Here: one key ("User-Agent") mapped to one value (the browser string). HTTP headers are literally just key-value pairs; Python dicts mirror them perfectly.

requests.get(URL, headers=HEADERS)

The star of the show. Sends an HTTP GET request to URL, with those headers attached. Returns a Response object — a Python object that holds everything that came back.

response.status_code

The HTTP status as an integer. 200 = good. 404 = not found. 403 = blocked. 500 = server exploded. Your first line of defense: always check this.

response.text

The raw HTML body of the response, as a Python string. 145,000 characters of markup. Next chapter, we'll slice it up with BeautifulSoup.

len(response.text)

len() is a built-in function that returns the length of a thing — strings, lists, anything "countable." Here: how many characters are in the HTML.

response.text[:500]

A slice — the first 500 characters of the string. [start:end] notation. We're just sampling so we don't flood the terminal with 145K chars.

🧠 Brain Power
Change the last line to print(response.text[:5000]) and run again. What's different? Why does that matter for building our scraper?

You'll see way more HTML — including, if you scroll through, some of the company-title-primary hooks we identified in Ch 01.

Why it matters: this proves the data we need is in response.text. It's a plain Python string. Our job from here is just "find the right substrings and pull them out." That's all BeautifulSoup is doing — smart string extraction with HTML awareness.

👉 The HTML is just a big string. The scraper is just careful slicing.

what broke?
Chapter 02.5 // When it breaks

💥 If something goes wrong

Python errors look scary but they're very honest. Here's the greatest hits:

❌ ModuleNotFoundError: No module named 'requests'

Meaning: Python can't find the requests library.
Fix: You probably ran the script without activating venv. Run source venv/bin/activate first, then pip install requests.

❌ Status: 403

Meaning: Cloudflare blocked you.
Fix: Check your User-Agent string. Make sure HEADERS is being passed to requests.get(). Without a proper UA, you look like a bot.

❌ ConnectionError / Timeout

Meaning: Network hiccup, or site is down.
Fix: Retry. Add timeout=10 to your requests.get() so it doesn't hang forever.

❌ IndentationError

Meaning: Python is strict about indentation. You probably mixed tabs and spaces.
Fix: Retype the indented lines. VS Code's bottom-right shows "Spaces: 4" — keep it consistent.
