the concepts
Concept 01 // Remote Control

🔐 SSH — your remote keyboard

When you "deploy" a THM machine, it spins up somewhere in the cloud. You need a way to type commands into THAT machine from YOUR machine. That's SSH.

📝 Think of it like WhatsApp voice call, but for shell commands

You pick up your phone (terminal), dial a number (IP address), and the OTHER phone (the target machine) lets you talk. Only here, instead of talking, you're typing commands — and the other machine executes them and sends the output back.

The whole conversation is encrypted end-to-end. That's what the "S" in SSH means — Secure Shell.

The command you'll run a thousand times

# Generic pattern ssh <username>@<target-ip> # This room specifically ssh tryhackme@<MACHINE_IP> # password: tryhackme
💡 Why port 22? SSH runs on TCP port 22 by default. Every scan you do as a pentester, port 22 being open is a huge deal — it means remote shell access is on the table if you can find creds or a vuln.
🧠 Brain Power
Why would a company shut off password-based SSH and force "key-based" auth instead?

Because passwords can be brute-forced. Attackers run tools like Hydra that hammer SSH with millions of password guesses per minute. An SSH key is a 4096-bit file living on your machine — mathematically infeasible to guess.

This is also why every CTF/bounty target with port 22 + password auth is a potential front door. 🚪

Concept 02 // Unlocking Commands

🚩 Flags, switches & the man page

Every Linux command has a default behavior AND a bunch of hidden modes you unlock with flags (a.k.a. switches, a.k.a. options).

📝 Think of flags like video game cheat codes

Default ls shows you some files. But ls -a? Shows you the hidden files too. ls -lh? Shows permissions + human-readable sizes. ls -la? Combines both.

Same command, different superpowers depending on which flags you pass.

Two ways to learn a command's flags

⚡ Quick reference: --help

Short summary of flags. Fast. Good when you just need to remember which flag does what.

ls --help

📖 Deep dive: man

Full manual — synopsis, all flags, examples, exit codes. Opens in a pager.

man ls
🎹 Man page navigation cheat sheet:
  • ↓ / ↑ — scroll line by line
  • Space / b — page down / page up
  • /pattern — search forward
  • n / N — next / previous match
  • q — quit (the ONE key every newbie forgets 😅)
Concept 03 // Moving Stuff Around

📂 Create, copy, move, delete

Six commands that you'll type more than your own name.

touch
touch
Create an empty file. Also updates timestamps.
touch note.txt
mkdir
make directory
Create a new folder.
mkdir mydir
cp
copy
Duplicate a file. Source → destination.
cp note note2
mv
move
Move OR rename. Same command either way.
mv note2 note3
rm
remove
Delete. Forever. No undo, no trash bin.
rm note  |  rm -R dir
file
file
Tell you what a file actually IS, regardless of extension.
file mystery_file
⚠️ rm is permanent. There is no recycle bin on Linux. rm -R / with root privileges will delete your entire system before you can blink. Many a sysadmin has learned this the painful way. Read twice, hit Enter once.

The file command is a secret weapon

An attacker uploads a file called image.jpg. Is it really a JPG? Or is it a PHP shell with a fake extension? file image.jpg reads the actual bytes (magic number) and tells you the truth.

tryhackme@linux2:~$ file image.jpg image.jpg: PHP script, ASCII text # 🚨 lying about its extension
Concept 04 // The Boss Fight

🛡️ Linux permissions decoded

This is the concept that confuses every single Linux beginner. We're going to murder that confusion in the next 3 minutes.

📝 Think of every file as a nightclub with 3 guest lists

Every file on Linux has three doors:

  • 🧑‍💼 Owner door — for the file's creator
  • 👥 Group door — for members of a specific group
  • 🌍 Others door — for literally everyone else on the system

Each door has its own bouncer checking three things: can you READ, WRITE, or EXECUTE?

The rwx string

When you run ls -l, you see something like this:

# This string describes the permissions: -rwxr-xr--  1 talha talha 0 Apr 19 10:37 script.sh ───────── owner grp others

Read it in chunks of 3:

Owner: rwx

Can read, write, AND execute. Full control.

Group: r-x

Can read and execute. NO writing. The - is a "nope" slot.

Others: r--

Can only read. Can't run it, can't change it.

Now the scary part: numeric mode

Each letter has a value:

  • r = 4 (read)
  • w = 2 (write)
  • x = 1 (execute)

Add them up per group. Three groups = three digits. That's your chmod 755.

🧠 Brain Power
A website's upload folder has chmod 777. Why is that every pentester's favorite sight?

777 means everyone can read, write, and execute — including the webserver. So if you upload a PHP file (because others=rwx lets you write), the webserver can execute it when you visit the URL.

Boom — arbitrary code execution. Reverse shell inbound. Game over. 🎯

This is why lazy "just chmod 777 it" = the gift that keeps on giving for attackers.

Switching users with su

Got creds for another user? Slide into their account:

tryhackme@linux2:~$ su user2 Password: ********* user2@linux2:/home/tryhackme$ # Or better — with full login environment: tryhackme@linux2:~$ su -l user2 user2@linux2:~$ # now in user2's home dir, user2's PATH, etc.
Concept 05 // The Map

🗺️ Common directories (the pentester's treasure map)

Linux organizes itself into standard folders. Knowing what lives where = knowing where to loot first.

/etc

System configs. Where the OS stores settings for users, services, apps.

🎯 Contains /etc/passwd (users) and /etc/shadow (hashed passwords — readable = jackpot). /etc/sudoers lists who can sudo.

/var

Variable data. Logs, backups, spool files — stuff that changes as the system runs.

🎯 /var/log is where logs live. Attackers delete logs from here to cover tracks. Also check /var/www for web app source.

/root

Root's home. The one directory only root can normally enter.

🎯 If you land a shell as root, flags and config files live in /root/. THM/HTB flags are often here.

/tmp

Temporary junk. Wiped on reboot. World-writable by default.

🎯 Gold for attackers: drop your enumeration scripts, reverse-shell binaries, LinPEAS here without needing write perms elsewhere.

/home

User homes. Each user gets /home/<username>.

🎯 Check ~/.ssh/, ~/.bash_history, ~/.env files — devs leave creds everywhere.

/

The root of everything. Not to be confused with /root. This is the top of the filesystem tree.

🎯 Everything else branches off from here. cd / lands you here.
🧠 Mental model: Linux's filesystem is one giant upside-down tree. / is the trunk. Everything else (including USB drives, other disks) gets "mounted" INTO this tree as branches.
