GPG 🔑

GNU Privacy Guard. Public-key cryptography for signing, verifying, and encrypting.
It's why apt install doesn't trash your system with malware.

"Anyone can sign their name. Only I can sign with my key." ✍️
Chapter 01 // Definition

📖 What is GPG?

GPG (GNU Privacy Guard) is the free, open-source implementation of the OpenPGP standard. It uses public-key cryptography to prove authorship, verify integrity, and encrypt data between people or machines.

📝 Think of it like a wax seal on a medieval letter

Back in the day, kings stamped molten wax with a ring only they possessed. Anyone receiving the letter could verify "yes, this really came from the king" just by looking at the seal. Nobody could forge it without the ring. GPG does the same thing — except the "ring" is a private key (a huge number stored in a file), and the "verify" step is math instead of eyeballs.

The two keys

🔐 Private key

NEVER leaves your machine. Protected with a passphrase. Used to sign things and decrypt things sent to you.

🔓 Public key

Shared openly — on websites, keyservers, email signatures. Used by others to verify your signatures and encrypt stuff only you can read.

Chapter 02 // The two things GPG does

🔐 Signing vs encrypting

✍️ Signing = "I made this, untouched"

You use your private key to sign. Anyone with your public key can verify. A signature doesn't hide the content — it just proves authorship + integrity.

Used by: apt repos, Git commits, software releases, emails.

🔒 Encrypting = "Only you can read this"

You use the recipient's public key to encrypt. Only their private key can decrypt. The content becomes unreadable to everyone except the intended reader.

Used by: secure email (PGP/Mailvelope), encrypted backups, password manager exports.

💡 Both at once: Usually you sign then encrypt — so the recipient knows it came from you AND nobody else can read it. GPG can do this in one command with gpg --sign --encrypt.

Chapter 03 // Everyday GPG

🌍 You already use GPG (you just don't know it)

📦 apt install trusts signatures

Every package from Ubuntu's repos is signed with Canonical's GPG key. Your machine stores that public key and refuses anything signed differently — or not signed at all.

🐙 GitHub "Verified" badges

That green "Verified" next to a commit? You GPG-signed the commit with git commit -S, and GitHub matched the signature to your public key.

📩 Encrypted email (PGP)

Security researchers and journalists use GPG (or its extension, PGP) to send emails only the recipient can read. Think Snowden → journalists.

💾 Software release integrity

Open-source projects publish a SHA256SUMS file AND a .sig file. Verify with gpg --verify SHA256SUMS.sig SHA256SUMS → "Good signature from ..."

Chapter 04 // Basic commands

⌨️ The GPG mini-cheatsheet

# Generate a new keypair gpg --full-generate-key # List your keys gpg --list-keys # public keys gpg --list-secret-keys # private keys (sensitive!) # Export your public key to share gpg --export --armor your@email.com > pubkey.asc # Import someone else's public key gpg --import their_pubkey.asc # Verify a signed file gpg --verify file.sig file # Encrypt a file for a recipient gpg --encrypt --recipient their@email.com secret.txt # Decrypt a file sent to you gpg --decrypt secret.txt.gpg
🚨 OPSEC: Your private key + passphrase = your identity. Leak the key file (from ~/.gnupg/) and an attacker can impersonate you. Back it up offline. Use strong passphrases. Revoke immediately if you suspect compromise.
