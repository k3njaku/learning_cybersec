📖 glossary · linux · process identity

PID 🔢

A unique integer assigned to every running process on Linux.
PID 1 is the parent of everything. The rest are its descendants.

"Every process has a number. Kill the number, kill the process." 🎯
Chapter 01 // Definition

📖 What is a PID?

Process ID — a unique integer the Linux kernel hands to every running program. Used to refer to it in commands like kill, nice, strace, etc.

📝 Think of it like a hospital wristband

When you check into a hospital, they put a wristband on you with a unique number. Doctors refer to you by that number even if a hundred other "John Smiths" are checked in. On Linux, every running program gets the same treatment — one unique PID for as long as it's alive. The instant it exits, that PID can be reused by the next process to spawn.

Chapter 02 // How PIDs get assigned

🔢 The numbering rules

  • PID 1 is always the first process the kernel starts at boot. Today that's systemd.
  • PIDs increment sequentially. The 547th process gets PID 547.
  • Default max PID on 64-bit Linux: 4,194,304 (cat /proc/sys/kernel/pid_max).
  • When PIDs wrap around, the kernel reuses freed ones — so PID 547 today isn't PID 547 after a reboot.
  • Each process has a PPID (parent PID) — who spawned it. ps -ef shows it.
Chapter 03 // Viewing processes

🔎 ps, top, htop, pgrep

ps aux

Snapshot of every process with user, PID, CPU%, memory%, command. The pentester's #1 enumeration command.

top

Live dashboard — refreshes every 3 seconds. Sort by CPU, memory, PID. Quit with q.

htop

Prettier, colorful top. Tree view, mouse support, easier keyboard shortcuts. Not always pre-installed.

pgrep <name>

Find PIDs matching a process name. pgrep sshd → prints the SSH daemon's PID.

Killing a process

kill 1337 # polite SIGTERM — asks process to clean up kill -9 1337 # SIGKILL — kernel nukes it immediately, no cleanup kill -SIGSTOP 1337 # pause a process (suspend) kill -SIGCONT 1337 # resume a paused process killall nginx # kill every process named nginx
💡 SIGTERM vs SIGKILL: SIGTERM (signal 15, default) politely asks. The process can trap it, flush buffers, save state, then exit. SIGKILL (signal 9) cannot be caught — kernel just yanks the rug. Always try SIGTERM first; SIGKILL only when something is stuck.
Chapter 04 // PIDs in the pentest workflow

🥷 Why enumerate processes

After getting initial access, ps aux is your lens into what's running and as whom. You're hunting for:

  • Privileged services — databases, web servers running as root.
  • Credentials in command-line args — mysql -uroot -pHunter2 visible in ps output. Yes, really. Happens.
  • Binaries in weird paths — /tmp/something or /home/user/custom-service → probably a custom script that's ripe for abuse.
  • Users you didn't know existed — ps aux | awk '{print $1}' | sort -u.
