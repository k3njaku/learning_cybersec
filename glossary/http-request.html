HTTP Request 📨

The letter your browser writes to a server every time you visit a page.
Once you see its shape, every scraping and pentest trick makes sense.

"Dear server, please send me /textile-companies. Thx, Chrome."
Chapter 01 // The anatomy

🫀 What's in a request

GET /textile-companies HTTP/1.1 ← method + path + version Host: textilepages.com ← which site on this server User-Agent: Mozilla/5.0 ... ← who I am Accept: text/html,application/xhtml+xml ← what I can handle Accept-Language: en-US,en;q=0.9 Cookie: session_id=abc123 ← anything stored from before ← blank line ← (optional) body, only for POST/PUT etc.

📝 The 3 layers

1. Request line — verb, path, protocol version

2. Headers — key: value pairs (who, what, where, cookies...)

3. Body — optional, for POST / PUT / PATCH (form data, JSON, files)

Every tool — browser, curl, Python requests, Postman — builds this exact same structure.

the verbs
Chapter 02 // HTTP methods

🎯 The verbs you'll actually use

📥 GET

"Give me this resource." Safe, idempotent. Browsers default. Scrapers live here.

📤 POST

"Create / submit this." Login forms, contact forms, creating new DB entries.

✏️ PUT / PATCH

PUT = replace whole resource. PATCH = partial update. Mostly in REST APIs.

🗑️ DELETE

"Remove this resource." REST APIs. Bug bounty gold when access control is broken.

🧪 OPTIONS

"What methods are allowed here?" CORS preflight requests.

👤 HEAD

Like GET but returns only headers (no body). Great for "does this URL exist?" checks without downloading KBs.

Chapter 03 // See them live

👀 How to watch HTTP in the wild

  • DevTools Network tab (F12 in any browser) — every request your browser makes is logged with headers, method, status, size, timing.
  • curl -v <url> — verbose mode prints the full request and response on the terminal.
  • Burp Suite / ZAP — intercept, modify, replay. The pentester's scalpel.
  • Python + requests — you build them yourself, line by line.
