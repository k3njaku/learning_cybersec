User-Agent 🪪

A text string where your HTTP client introduces itself.
Set it wrong, and you get 403'd. Set it well, and you walk right in.

"Hi, I'm Chrome 120, please let me in."
Chapter 01 // Definition

📖 What is a User-Agent?

An HTTP request header where the client declares what software it is.

📝 Bouncer analogy

Every visitor to a website is asked by the bouncer: "Who are you?" Your browser answers with a User-Agent string:

Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36

Translation: "I'm Chrome 120 running on Linux." Looks weird, but that's 30 years of browser evolution fossilized into one line.

Python's requests library, if you don't override, answers: "I'm python-requests/2.31.0." Bouncer raises an eyebrow.

why it matters
Chapter 02 // Why it matters

💥 Scraping / pentest implications

🚫 The instant 403

Many sites + WAFs (Cloudflare, Akamai, AWS WAF) block default python-requests UAs on sight. It's the lowest-hanging bot signal.

🪪 Easy bypass

Setting User-Agent: Mozilla/5.0 ... gets through most basic bot checks. A one-line header = instant access.

🎭 Fingerprinting

Advanced WAFs combine UA + TLS fingerprint + JS execution to detect bots. A real UA alone isn't always enough — but it's always the first step.

🔀 Rotation

For big scraping jobs, rotate UAs across requests. Cycle Chrome / Firefox / Safari / mobile variants. Each request looks like a different person.

Chapter 03 // Code

🎯 How to set it in Python

import requests UA = "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 " \ "(KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36" r = requests.get("https://textilepages.com/textile-companies", headers={"User-Agent": UA}) print(r.status_code) # 200 ← rather than 403
☕ Find your own UA: open your browser, visit httpbin.org/user-agent. It echoes back whatever your current browser sends. Copy-paste it into your scraper.

🎯 Quick Check

Your Python scraper works via curl -A "Mozilla/5.0 ..." <url> but fails with requests.get(url) returning 403. Why?
Python is slower than curl.
Cloudflare blocks Linux.
You passed a fake UA to curl, but the Python call is using the default python-requests/... UA. Add a headers={"User-Agent": "Mozilla/5.0 ..."} dict to your request.
Your Python version is outdated.
