Brute Force 🔨

Trying millions of passwords until one works.
The least elegant attack, yet it works constantly.

"what if I just tried every password?" 🤔 — works more than you'd think.
Chapter 01 // Definition

📖 What is brute force?

A brute-force attack is exactly what it sounds like — trying every possible password (or a list of common ones) until one is accepted. No cleverness. Just raw iteration.

📝 Think of it like trying every combination on a bike lock

3-digit lock has 1,000 combinations (000–999). If you try them one by one at 1 per second, you'll crack it in under 17 minutes. No skill. Just patience and a working finger.

Computers try passwords the same way — except at millions per second.

🧠 Why it still works in 2026: Even though password policies exist everywhere, password reuse and weak defaults mean brute-force still compromises thousands of accounts daily. Mirai botnet became famous by just trying ~60 default creds on IoT devices.
Chapter 02 // Flavors of brute force

🎭 Different strategies

🎰 Pure brute force

Try literally every combination: aaaa, aaab, aaac.... Only works for short passwords. 8-char alphanumeric = 218 trillion tries = GPU cluster for days.

📖 Dictionary attack

Try a wordlist of common passwords first (password123, qwerty, letmein). Wordlists like rockyou.txt (14M entries) crack most weak passwords in seconds.

📜 Credential stuffing

Take leaked user:pass pairs from breach X, try them on site Y. Banks on password reuse. Stunningly effective.

💦 Password spraying

Try ONE common password (Spring2024!) against MANY usernames. Stays under per-account rate limits. Used heavily by APT groups.

🎯 Hybrid / rule-based

Take a wordlist, apply rules: add numbers, capitalize, swap letters. password → P@ssw0rd1, P@ssw0rd!. Tools like hashcat specialize in this.

🌈 Rainbow tables

Pre-computed hash lookups for stolen password databases. Not live brute-force — offline cracking. Defeated by salting.

Chapter 03 // The tools

🛠️ The brute-force arsenal

# Hydra — the classic, works against everything hydra -l admin -P rockyou.txt ssh://10.10.10.5 # Against HTTP login forms hydra -l admin -P pass.txt 10.10.10.5 http-post-form \ "/login:username=^USER^&password=^PASS^:F=invalid" # Burp Intruder (GUI, great for web) # Use "Sniper" or "Cluster bomb" attack type with the right payload set # ffuf — fast HTTP fuzzer, great for basic auth / login brute ffuf -w pass.txt -u "https://site.com/login" \ -X POST -d "user=admin&pass=FUZZ" \ -mc 200 -fs 0 # Hashcat — OFFLINE cracking of captured hashes hashcat -m 1400 hashes.txt rockyou.txt # SHA-256 hashcat -m 3200 hashes.txt rockyou.txt # bcrypt # John the Ripper — the OG hash cracker john --wordlist=rockyou.txt hashes.txt
⚠️ Legal note: Brute-forcing a system you don't own or have permission for = unauthorized access = potentially a crime (CFAA, Computer Misuse Act, etc.). Practice only on CTF platforms (THM, HTB), bug bounty programs in scope, or your own test environments.
Chapter 04 // Defense

🛡️ Killing brute-force dead

  • Rate limiting — N attempts per IP per minute. After that, slow down or block.
  • Account lockout — lock the account for 15 mins after 5 failed attempts. Attacker's automated tool hits a wall.
  • CAPTCHAs — after 3 failures, require hCaptcha/reCAPTCHA. Humans still proceed; bots die.
  • MFA — even if the password is correct, attacker doesn't have the 2FA code. Kills credential stuffing cold.
  • Block known-bad passwords at signup. Use HaveIBeenPwned Passwords API (free).
  • Alert on spikes — sudden surge of 401s from one IP range? Automated block.
  • Long passwords required — 14+ chars with diversity. Brute force becomes computationally unfeasible.
