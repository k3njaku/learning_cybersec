requests 🌐

The Python library that turns "talk to a website" into a single line of code.
If you write one scraper, you'll type import requests a thousand times.

"HTTP for humans."
Chapter 01 // Definition

📖 What is requests?

A third-party Python library that makes HTTP calls stupid-easy. Not built into Python — you pip-install it.

📝 Phone call analogy

Making an HTTP request manually is like building a telephone from wire and a battery just to make a call. requests is a phone with a dial pad.

You tell it who to call (URL) and what to say (headers/data) — it handles every ugly detail underneath.

import requests r = requests.get("https://textilepages.com/textile-companies") print(r.status_code) # 200 print(len(r.text)) # 145703
why it matters
Chapter 02 // Why it matters

💥 Why pros reach for it first

🐍 Beautifully Pythonic

Everything is a method or attribute — no callbacks, no weird async unless you want it. r.text, r.json(), r.status_code. Done.

🍪 Sessions & cookies

requests.Session() persists cookies across calls — crucial for sites behind login walls or CSRF tokens.

🪪 Custom headers trivial

Pass a dict. Rotate User-Agents, add Authorization bearer tokens, set Referer headers. Any HTTP trick fits in one line.

🌐 Universal

The same library works for scraping, for calling REST APIs, for cybersec tools (recon scanners, custom brute-forcers, SSRF payload testers). One skill, many paychecks.

the 5 moves
Chapter 03 // The 5 moves you'll use forever

🎯 Cheat sheet

# 1. Simple GET r = requests.get("https://example.com") # 2. GET with headers (User-Agent, Auth, etc.) headers = {"User-Agent": "Mozilla/5.0 ..."} r = requests.get(url, headers=headers) # 3. GET with query parameters → example.com/search?q=python&page=2 r = requests.get("https://example.com/search", params={"q": "python", "page": 2}) # 4. POST with form data (login forms, submissions) r = requests.post(url, data={"username": "admin", "password": "hunter2"}) # 5. Session (keeps cookies between calls — for logins, multi-step flows) s = requests.Session() s.post(login_url, data={"u": "a", "p": "b"}) r = s.get(protected_url) # session cookie sent automatically
gotchas
Chapter 04 // Stuff that bites you

⚠️ Gotchas

  • Default UA is python-requests/x.x.x — sites see that and say "bot." Always override with a realistic User-Agent.
  • No timeout by default → your script can hang forever. Always: requests.get(url, timeout=10).
  • r.text vs r.content: text is decoded to string (good for HTML), content is raw bytes (good for binary — images, PDFs).
  • For JSON APIs, use r.json() — returns a dict directly, no manual parsing.
  • Not built-in. You MUST pip install requests inside your active venv.

🎯 Quick Check

Your script calls requests.get("https://api.example.com/data") — how do you check if the call succeeded before parsing?
Check if r.text is empty.
Check r.status_code == 200 (or use r.raise_for_status()).
Time the call with a stopwatch.
Run it twice and compare.
