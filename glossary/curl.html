Chapter 01 // Definition

📖 What is curl?

curl (Client URL) is a command-line tool that transfers data using URLs. It supports HTTP(S), FTP, SMTP, IMAP, POP3, and ~20 other protocols. For web pentesters, it's the go-to for making raw HTTP requests.

📝 The sniper rifle analogy

A browser is a shotgun — one trigger, 47 invisible actions (cookies, cache, scripts, trackers, redirects, form auto-fills).

curl is a sniper rifle — exactly one HTTP request, exactly as you specify. Nothing hidden, nothing auto-magic.

That precision is why every bounty writeup with a POC uses curl.

The canonical command shape

curl [options] URL

Chapter 02 // The 10 flags that cover 90% of usage

🚩 Essential flags

Flag What it does
-X METHODHTTP method (GET, POST, PUT, DELETE, PATCH)
-H 'Header: val'Add a custom header. Use repeatedly for multiple headers.
-d 'data'Request body. Implies POST. Use -d @file to load from file.
--data-urlencodeLike -d but URL-encodes values. Safer for special chars (', &, =).
-iInclude response headers in output. Critical for auth / redirect debugging.
-IHEAD request — only response headers, no body.
-LFollow redirects. Otherwise curl stops at the first 3xx.
-sSilent — suppress progress bar. Use in scripts.
-c / -b FILE-c saves cookies, -b sends them. Combine to maintain sessions.
-kIgnore SSL errors. Useful for self-signed certs in lab setups.

Chapter 03 // Copy-paste recipes

🍳 The recipes you'll actually use

1. Plain GET

curl https://example.com/api/users

2. GET with a query string (safe encoding)

curl -G "https://site.com/search" \ --data-urlencode "q=some value with spaces & symbols"

The -G converts --data into a query string instead of a POST body.

3. POST JSON (API testing)

curl -X POST https://api.site.com/login \ -H "Content-Type: application/json" \ -d '{"username":"admin","password":"x"}'

4. POST form-encoded (login forms)

curl -X POST https://site.com/login \ --data-urlencode "username=admin" \ --data-urlencode "password=hunter2"

5. Session-aware requests (cookies)

# Step 1: log in, save cookies curl -c cookies.txt -X POST site.com/login -d "u=x&p=y" # Step 2: reuse those cookies on authed endpoints curl -b cookies.txt site.com/account

6. Send a custom header (auth tokens, bypass WAFs)

curl -H "Authorization: Bearer eyJhbGc..." \ -H "X-Forwarded-For: 127.0.0.1" \ https://api.site.com/admin

7. Quick status-code probe (scripting)

curl -s -o /dev/null -w "%{http_code}\n" https://site.com/admin

Prints just the HTTP code. Perfect for bash loops that probe many URLs.

Chapter 04 // curl for hacking

🥷 Why every pentester lives in curl

🎯 SQLi testing

Craft exact payloads with --data-urlencode — no browser auto-encoding weirdness. Reproduce findings with one line.

🔀 Header injection + spoofing

Set any Host, X-Forwarded-For, Referer, User-Agent. Test auth bypasses, cache poisoning, SSRF.

🍪 Session replay

Copy a real browser cookie, paste into -H "Cookie: ...", replay any request as that user.

🔁 Bulk probing

Bash loops + curl = fast enumeration: test 100 IDs in a loop looking for IDOR, count columns for a UNION attack, brute-force directories.

🧠 Brain Power
You send a curl POST to a login form with the correct credentials. Response is 302 redirecting to /dashboard. You curl /dashboard — get 401 Unauthorized. Why?

You forgot the session cookie. The login response set a Set-Cookie: header, but your next curl call didn't send it back. Two fixes:

① Use -c cookies.txt on the login request + -b cookies.txt on subsequent requests.
② Use -L on the login request to auto-follow the redirect (curl maintains cookies across a single invocation's redirect chain).
