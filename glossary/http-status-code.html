HTTP Status Code 🚦

A 3-digit number the server sends with every response to tell you what happened.
Every scraper, every pentester, every web developer reads these like mood rings.

"200 means 'yes.' 403 means 'no, go away.' 500 means 'I'm on fire.'"
Chapter 01 // The 5 families

📖 Status codes by first digit

1️⃣ 1xx — Informational

"Hold on, still thinking." You rarely see these. Example: 101 Switching Protocols (for WebSockets).

2️⃣ 2xx — Success ✅

200 OK (here you go), 201 Created (new resource), 204 No Content (done, no body). 200 is your happy place.

3️⃣ 3xx — Redirect 🔀

301 Moved Permanently, 302 Found, 307 Temporary Redirect. Server says "the thing you want is somewhere else." Requests follows them by default.

4️⃣ 4xx — YOUR fault ❌

400 bad request, 401 unauthorized, 403 forbidden, 404 not found, 429 too many requests. Scrapers see these a lot.

5️⃣ 5xx — SERVER'S fault 🔥

500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable, 504 Gateway Timeout. Retry later — it's not you.

🕵️ The coffee joke

418 I'm a teapot. Real code. RFC 2324. Servers return this when... they're teapots. Honestly.

Chapter 02 // What each code means for YOU

🎯 Scraper/pentester handbook

✅ 200 OK

You got it. Parse the body. Proceed.

🔀 301 / 302

Page moved. requests auto-follows by default. Check r.url to see the final destination.

🚫 403 Forbidden

You're blocked. Most common causes in scraping: (1) bad / missing User-Agent, (2) WAF / Cloudflare rule, (3) missing cookies / session, (4) IP on a blocklist.

🔍 404 Not Found

Typo in URL, or resource deleted. Not a bot signal. Fix the URL.

⏳ 429 Too Many Requests

Rate limit. Slow down (time.sleep()), rotate IPs / UAs, respect Retry-After header. Classic scraper wall.

🔥 5xx

Server problem. Retry with backoff. If it persists → site might be genuinely down.

Chapter 03 // Code

🧑‍💻 Checking it in Python

import requests r = requests.get(url, headers=headers, timeout=10) if r.status_code == 200: print("Got it") elif r.status_code == 429: print("Slow down. Retry-After:", r.headers.get("Retry-After")) elif r.status_code >= 500: print("Server broke, try again later") else: print("Unexpected:", r.status_code)

🎯 Quick Check

Your scraper hits 30 URLs quickly and starts getting 429 responses. What's happening and what's the right fix?
The site is down. Wait an hour.
You're rate-limited. Add delays (time.sleep), respect Retry-After, and consider exponential backoff. Also: are you hitting the same server too fast?
You got banned forever.
Switch to HTTPS.
