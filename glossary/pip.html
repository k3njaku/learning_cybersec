Python's App Store. One command puts any of 500,000+ libraries on your machine.

"pip install <thing> → thing is yours."
Chapter 01 // Definition

📖 What is pip?

Python's official package installer. It talks to the Python Package Index (PyPI — pypi.org) and downloads/installs any published library.

📝 App Store analogy

PyPI = the App Store. pip = the button that installs. Hit the button, the app shows up in your phone (Python).

You'll use it forever. Every Python dev uses it daily.

Chapter 02 // The verbs

🎯 Commands you'll use 99% of the time

# Install a single package pip install requests # Install multiple at once pip install requests beautifulsoup4 lxml # Install a specific version pip install requests==2.31.0 # Install from a requirements file (shared dependencies) pip install -r requirements.txt # What's installed in my current (venv) environment? pip list # Upgrade a package pip install --upgrade requests # Remove a package pip uninstall requests # Export current packages to a file (for git / teammates) pip freeze > requirements.txt
rules
Chapter 03 // The rules

🛡️ Habits that save pain

✅ Always activate venv first

Then pip install puts stuff in the right place. No venv → global Python → future conflicts.

✅ Pin versions for projects you ship

pip freeze > requirements.txt. Now anyone (or future-you) can run pip install -r requirements.txt and get the exact same environment.

🚫 Don't use sudo pip

Installing as root into the system Python is a classic way to brick your OS's Python tools. If you see an externally-managed-environment error — that's the signal to use a venv.

🕵️ Watch for typosquats

Attackers publish packages like requessts (extra s) hoping you misspell. Double-check the name before installing.

🎯 Quick Check

You clone a Python project from GitHub that has a requirements.txt — what's the magic command to set up its dependencies?
