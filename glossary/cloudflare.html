Cloudflare ☁️

A massive CDN + DDoS shield + WAF sitting in front of ~20% of the web.
When you scrape or pentest, you meet Cloudflare before you meet the actual server.

"The internet has a bouncer. His name is Cloudflare."
Chapter 01 // Definition

📖 What is Cloudflare?

A reverse-proxy service. Your traffic goes to Cloudflare's network first, which decides what to do with it, then (maybe) forwards it to the actual origin server.

📝 Nightclub analogy

The origin server is the VIP room. You (the user) want in. Cloudflare is the bouncer at the door checking every visitor:

  • Looks like a bot? → CAPTCHA or block.
  • Looks fine? → Cache the previous response and hand it to you (fast).
  • Weird / cached out? → Forward to the origin and bring back the answer.

Good for the website: speed, DDoS protection, blocking obvious bots. Good for you as a visitor too — usually invisible.

how to spot it
Chapter 02 // Recon signals

🔍 How to tell a site is behind Cloudflare

  • Response header server: cloudflare
  • Response header cf-ray: <hash>-<datacenter> (unique ID per request)
  • Response header cf-cache-status: (HIT / MISS / DYNAMIC)
  • Cookies like __cf_bm (bot management), cf_clearance (JS challenge solved)
  • "Checking your browser before accessing…" interstitial page
  • 403 with a CF-branded error page
# Quick check from the terminal: curl -sI https://example.com | grep -i cf
friend or foe?
Chapter 03 // Impact on scraping / pentesting

⚖️ When it matters, how much it matters

😌 Lazy-mode Cloudflare

Just a CDN in front — no bot-management enabled. You'll see server: cloudflare but a normal User-Agent walks right in (like textilepages.com did for us). Basic requests code works.

🛡️ Bot Fight Mode / Managed Challenge

Serves a JS challenge. Your Python script gets HTML that says "Checking your browser…" — not the real page. Need browser automation (Selenium/Playwright) or services like FlareSolverr / cloudscraper.

🧩 CAPTCHA mode

Turnstile (Cloudflare's CAPTCHA). Hardest to bypass. Options: solve manually + reuse cf_clearance cookie, use anti-CAPTCHA services, or give up / find an API.

🔎 Origin discovery (pentest)

If you can find the origin server's real IP (via Shodan, Censys, historical DNS, SSL cert transparency logs), you bypass Cloudflare entirely and hit the origin directly. Classic recon move.

☕ Rule of thumb: if your request returns real HTML with a 200, Cloudflare is letting you through. If you get a 403, a 503 with a JS challenge, or a "Just a moment…" page, you're being checked. Diagnose with curl -v first.

🎯 Quick Check

Your scraper gets 200 OK but the body is a short HTML page that says "Checking your browser…" — what's happening?
The server is still loading.
Cloudflare served a JavaScript challenge page instead of the real content. Your script needs to solve the JS (via a headless browser or a tool like FlareSolverr) and reuse the resulting cf_clearance cookie.
You hit rate limits.
Your User-Agent is too long.
