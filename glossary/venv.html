venv 📦

A tiny private Python kitchen for each project.
Without it, your laptop becomes a junk drawer of conflicting library versions.

"One project, one kitchen."
Chapter 01 // Definition

📖 What is venv?

Python's built-in module for creating an isolated Python environment — a folder that contains its own Python interpreter copy and its own installed packages.

📝 The kitchen analogy

Imagine 10 chefs sharing one kitchen. Every chef needs a different set of knives, a different fridge temperature, a different oven settings. Chaos.

A venv gives each chef (each project) their own kitchen. Install whatever libraries at whatever versions you need. None of it leaks into your other projects or your OS.

# Create a venv called "venv" inside the current folder python3 -m venv venv # Activate it (Linux/macOS) source venv/bin/activate # Your prompt now shows (venv) — kitchen lights are on # Install libraries — they go INSIDE this venv only pip install requests beautifulsoup4 # Leave the kitchen when you're done deactivate
why it matters
Chapter 02 // Why you always do this

💥 The mess without venv

💣 Version hell

Project A needs requests 2.25. Project B updates to requests 2.31 and breaks something in A. You spend Saturday debugging. All because of one pip install.

🧹 Clean deletes

Done with a project? Delete its folder. Everything it ever installed goes with it. No orphan libraries, no system cruft.

📦 Shareable deps

pip freeze > requirements.txt inside a venv produces a clean list of exact versions. Ship that file, anyone can recreate your environment.

🛡️ Ubuntu now forces it

Modern Debian/Ubuntu blocks global pip install by default (externally-managed-environment). This is Ubuntu saving you from yourself. Use venv.

🎯 Quick Check

You run pip install requests but forget to activate your venv first. What's the cleanest fix?
Restart your computer.
Activate the venv (source venv/bin/activate), then run pip install requests again. The install into the venv overrides nothing, and optionally pip uninstall requests globally to clean up.
Delete Python and reinstall.
Use sudo.
